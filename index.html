Lightseeker

unreal engine 5.0.3
c++
blueprints

A third-person 3D Action Role-Playing Game (ARPG) developed as my final year university capstone project. Built for Windows using Unreal Engine 5, the project was developed collaboratively with one teammate, allowing for a clear division of responsibilities. I was solely responsible for the complete game development lifecycle — from core gameplay systems to polish — while my teammate focused on documentation, research, and supporting academic deliverables. This separation enabled me to fully dedicate my time to designing, developing, and refining the game systems. My responsibilities included:

  • • Gameplay Programming.
  • • Player Movement & Attacks.
  • • Enemy Movement & Attacks.
  • • Animation Mapping.
  • • UI Design.
  • • Sound Integration.
  • • Game Mechanics. (Health, Stamina, Pick ups etc.)

The project was implemented primarily in C++, ensuring optimal control over character behavior and system performance. Blueprints were selectively used for the boss battle and visual scripting of event-driven mechanics. The boss AI features a dynamic projectile system where the enemy tracks and throws swords at the player, adjusting based on player movement and state.
Through this project, I honed my skills in systems design, animation pipelines, AI behavior, and gameplay loop structuring. The successful completion of Lightseeker also increased my ability to independently build and polish a full-fledged game experience within academic and technical constraints.

BlueprintAR

unity
c#
augmented reality
python
fastAPI
tesseract OCR
opencv

An experimental augmented reality application built to explore the integration of computer vision, backend processing, and AR-based visualization. The app allows users to scan or upload room layout images (e.g., floorplans or architectural sketches), which are then processed via a Python backend using OCR to extract dimensional data. That data is returned to Unity as JSON and used to generate virtual room structures within an AR scene. This project was developed independently as a learning exercise to deepen my understanding of real-time AR workflows, API integration, and automated environment generation using data extracted from images.I was responsible for the following:

  • • Backend Integration with Unity.
  • • Image Input Handling.
  • • AR Implementation.
  • • Transform Controls.


While the application was not built for production, it served as a valuable exercise in coordinating multiple systems: Unity’s AR plane detection and rendering, Python-based server logic, OCR integration, and JSON-driven procedural generation. It also provided real-world experience in designing lightweight communication between game engines and backend services. Though the walls are not present in the video, I did later add the walls to the floor prefab.
